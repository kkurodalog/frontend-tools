## 1. 機能テスト

| # | 観点 | 合格条件 |
|---|---|---|
| 1-1 | 投入（D&D / ファイル選択 / フォルダ一括） | 3経路すべてで画像が投入でき、リストに表示される |
| 1-2 | 種別の一括選択 | 全画像に一括で種別を当てられ、全行に反映される |
| 1-3 | 種別の個別変更 | 1枚ごとに種別プルダウンで変更でき、その画像だけ再圧縮される |
| 1-4 | 6種別プリセットの圧縮結果 | 各種別が `01_spec.md` §1 の目標フォーマット・品質値・リサイズ上限どおりに出力される（①強圧縮/②画質優先/③中圧縮/④減色/⑤⑥可逆） |
| 1-5 | 手動オーバーライド（品質/フォーマット/リサイズ） | 各値を変更すると出力に反映され、フォーマット変更で品質既定が種別別既定に追従する |
| 1-6 | Before/After 容量・削減率 | 全画像で元容量・出力容量・削減率が正しく表示される |
| 1-7 | 劣化確認（並置・必須） | Before/After が並置表示され、重要写真の画質を目視確認できる |
| 1-8 | ルーペ（任意） | 実装した場合、拡大位置が正しく表示される（未実装なら対象外と明記） |
| 1-9 | フォーマット別容量比較（任意） | WebP vs AVIF vs 元 の容量が並び、種別判断の参考になる |
| 1-10 | EXIF/メタ削除＋Orientation正立（D3） | 出力にEXIFが残らず、回転情報を持つ画像が正立して出力される（倒れない） |
| 1-11 | 元形式維持＝穏やかJPEG最適化（D7） | リサイズ＋メタ削除＋q≈82で再エンコードされ、ほぼ無劣化で軽量化される |
| 1-12 | 自動種別推測＋ON/OFFトグル（D2） | ON で初期種別が割当され根拠が表示される／OFF で推測されない／推測後も手動で直せる |
| 1-13 | ⑤ SVG 助言バナー（D5） | ⑤割当時に「SVGにできませんか?」助言が表示される |
| 1-14 | 個別DL / 一括ZIP DL | 個別DLと一括ZIPが動作し、ファイル名が元名維持＋拡張子追従（D10）になっている |
| 1-15 | HEIC 非対応案内（D1） | HEIC投入時、当該1枚だけスキップされ「事前にJPEG化を」案内が出る（全体は止まらない） |
| 1-16 | エラー耐性 | 破損ファイル・非対応形式・上限超過は当該1枚だけスキップ＋メッセージ／他の処理は継続する |

## 2. ブラウザ対応

| # | 観点 | 合格条件 |
|---|---|---|
| 2-1 | 最新 Chrome / Edge | WASM デコード・エンコード（WebP/AVIF/PNG可逆）が全種別で動作 |
| 2-2 | 最新 Safari | WebP/AVIF エンコードが動作（Safari の AVIF 対応状況を確認・不可なら WebP フォールバック挙動が正しい） |
| 2-3 | 最新 Firefox | WASM・Web Worker が動作し、AVIF も処理できる |
| 2-4 | 非対応環境フォールバック | WASM/Web Worker 非対応時に「お使いのブラウザでは利用できません」等のメッセージが出る（白画面にならない） |

## 3. パフォーマンス（大量画像時の固まり）★最重要リスク

| # | 観点 | 合格条件 |
|---|---|---|
| 3-1 | 20〜30枚一括（WebP） | 一括処理中もUIが固まらず、進捗バーが進む |
| 3-2 | AVIF 含む一括（★固まり対策の核） | Web Worker 逐次処理が効き、AVIFを含む20枚一括でもメインスレッドが凍結しない／「時間がかかります」注記が出る |
| 3-3 | 進捗表示 | n/N 枚＋処理中ファイル名が表示され、無反応に見えない |
| 3-4 | メモリ上限ガード（D8） | 〜25MB超の1枚はスキップ、枚数上限超過時に警告が出てフリーズしない |
| 3-5 | WASM 遅延ロード | AVIF/oxipng/imagequant を使うまで重いWASMがDLされない（初回操作が速い） |

## 4. アクセシビリティ（WCAG AA）

| # | 観点 | 合格条件 |
|---|---|---|
| 4-1 | キーボード操作 | 投入（ファイル選択ボタン）・種別変更・実行・DL がキーボードだけで完結する |
| 4-2 | D&D 代替手段 | D&D できない環境でもファイル選択ボタンで全機能が使える |
| 4-3 | フォーカス可視 | 全操作要素にフォーカスリング（共通枠 `--color-focus-outline`）が出る |
| 4-4 | コントラスト | テキスト・UI要素が AA 比率を満たす（ライト/ダーク両テーマ） |
| 4-5 | スクリーンリーダ | 種別・容量・削減率・進捗・エラーが読み上げられる（aria-live で進捗/結果を通知） |
| 4-6 | 言語不問UI（U-05） | アイコン＋数値主導で、種別・容量・削減率・進捗が言語に依存せず識別できる |

## 5. レスポンシブ

| # | 観点 | 合格条件 |
|---|---|---|
| 5-1 | モバイル（〜767px） | ドロップゾーン・画像リスト・並置プレビューが縦積みで崩れず操作できる |
| 5-2 | タブレット〜デスクトップ | 768px 以上で共通枠（ナビ）とツール本体が整合（共通枠の768px境界＝design-concept.md） |
| 5-3 | clamp/トークン準拠 | 余白・タイポが clamp＋space スケールに乗り、ビューポート間で破綻しない |
| 5-4 | iOS 入力ズーム回避 | フォーム入力が16px維持でフォーカス時にズームしない（design-concept §2-2） |

## 6. 規範準拠（Reid 照合）

| # | 観点 | 合格条件 |
|---|---|---|
| 6-1 | FLOCSS 命名 | クラスが FLOCSS（接頭辞・層）規範に準拠し、共通枠のクラス体系と整合 |
| 6-2 | デザイントークン | 色・余白・タイポが本体派生トークン（tokens.css）を使い、独自色を作っていない（design-concept §2-2 のツール固有トークン `--tool-panel-*`/`--dropzone-*`/`--field-*` を使用） |
| 6-3 | clamp レスポンシブ | 直値の場当たり指定でなく clamp＋space スケールに乗っている |
| 6-4 | static-template 基盤整合 | `dev/` のテンプレ基盤（SCSS glob import・`_viewport.js`・`<head>` link クリティカルパス）に乗り、FOUC が起きない |
| 6-5 | 共通枠との一貫性 | ツール固有UIが共通枠（header/footer/パンくず/制作者カード/CTA）の世界観（b-tech-cool・768px境界・ダーク追従）に乗っている |
| 6-6 | プライバシー表記 | 「画像を送信しない」がファーストビューに明示されている（差別化メッセージ） |
