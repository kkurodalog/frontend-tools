## 1. このツールは何か（一言定義）

**画像をその役割（種別）で仕分けし、種別ごとに最適な圧縮方針を自動で当て、かつ手動で選び直せる、ブラウザ内完結（WASM）の画像最適化ツール。** 画像は一切サーバーに送信しない。

「全部おまかせ」で済ませたい人にはおまかせ（自動種別推測）を、種別ごと・1枚ごとに細かく決めたい人には手動オーバーライドを——**「選べる自由」と「おまかせ」の両立**が設計の核。

## 2. 解決する痛点

TinyPNG 型の「全部おまかせ・圧縮率も方式も選べない」一律圧縮では、**容量を最優先で強く潰したい背景画像と、劣化させたくない顔写真を、同じ強さで処理してしまう**。

本ツールは画像を**役割（種別）で仕分け**し、種別ごとに最適な圧縮方針（目標フォーマット／品質値／リサイズ上限）を当てることで、この痛点をピンポイントで解消する。「背景は強く・重要写真は画質優先・一般写真は中圧縮」を、画像をドロップして種別を選ぶだけで実現する。

## 3. ターゲット（PLAN.md §2-8 positioning に準拠）

| 区分 | 誰 | ニーズの中心 |
|---|---|---|
| **主**（スコープを決める物差し） | フロント制作者（黒田さん本人を含む） | 毎案件・毎ページの画像書き出し直後に、種別ごとに圧縮を選び分けたい・自分のワークフローで毎回使える速さ |
| **従**（検索流入の間口） | 学習者・ブロガー | 「背景と顔写真を一律で潰したくない」潜在不満／プライバシー（送信しない）／無料・無制限 |

> 主＝自分用のドッグフーディングが選定動機。**主のために設計し、従に配慮する**順序を崩さない（迷ったら主の作業効率を優先）。プロダクト整合の判断軸＝「**この機能はフロント/Web制作の用途に必要か?**」（PLAN.md §2-8）。

## 4. 差別化（競合＝TinyPNG 等との違い）

1. **ブラウザ内 WASM 完結＝画像を送信しない（プライバシー訴求）** — TinyPNG はアップロード型。本ツールは画像が端末から出ないため、未公開素材・受領素材・スクショ（個人情報を含みうる）でも安心して処理できる。サーバー不要＝無料・無制限・回数制限なし。
2. **種別ごとに選べる自由＋おまかせの両立** — 一律圧縮ではなく、6種別プリセットで圧縮方針を選び分けられる。さらに自動種別推測（ON/OFF切替可）で「おまかせ」も、1枚ごとの手動オーバーライド（品質／フォーマット／リサイズ）で「玄人の微調整」も両立する。Squoosh の自由度を、プリセットで簡単にした位置づけ。
3. **劣化確認で重要写真の品質を担保** — Before/After を並置し、重要写真の画質低下を目視で確認してから書き出せる。

## 5. v1 スコープ（やる／やらない）

### やること（U-01＝案②フル / D1〜D10 確定済み）

- 6種別プリセット（①背景・装飾 ②写真(重要) ③写真(一般) ④イラスト・図版 ⑤アイコン・ロゴ ⑥スクショ・UI）⇄ パラメータ自動当て
- 出力フォーマット: WebP（可逆/非可逆）／AVIF／PNG可逆（oxipng）／元形式維持（穏やかなJPEG最適化）
- 自動種別推測（解像度・透過・色数から推測 ＋ ON/OFFトグル）／手動オーバーライド（種別変更・品質・フォーマット・リサイズ）
- EXIF/メタデータ削除（デフォルトON）＋ Orientation 正立焼き込み
- ④イラストの減色量子化（imagequant）／⑤アイコンの「SVGにできませんか?」助言
- Before/After 容量・削減率・劣化確認（並置必須・ルーペ任意）／フォーマット別容量比較／個別DL・一括ZIP DL
- 入力: PNG / JPEG / WebP（GIF静止画・AVIF入力は任意）／複数枚一括・D&D・ファイル選択／フォルダ一括（任意）
- Web Worker 逐次処理（AVIF のブラウザ固まり対策）／進捗表示／枚数・サイズ上限ガード

### やらないこと（スコープ外・YAGNI排除）

画像編集（トリミング/フィルタ/補正/加工）・画像生成AI/背景透過AI・アップロード保存/クラウド同期/共有リンク・アカウント/ログイン・**アニメGIF/動画の最適化**・URL逆引き/既存ページ画像の取得。

加えて v1 では入れない（将来候補）: **HEIC入力**（ブラウザデコード不可・バンドル増／「事前にJPEG化を」案内で代替）・SVG自体の最適化（SVGO相当・⑤では助言のみ）・クリップボード貼付・プリセットのカスタム保存・スライダー重ね比較・MozJPEG極限圧縮・PWA/オフライン化。

> 一線＝**「最適化に集中し、編集も保存もしない」**。

## 6. 成功基準

- **主痛点の解消（最重要）**: 黒田さん自身の3つの痛点——背景=強圧縮 / 重要写真=画質優先 / 一般写真=中圧縮——が、画像をドロップして種別を選ぶだけで満たされること。
- **選べる自由**: 種別変更・1枚ごとの品質/フォーマット/リサイズの手動オーバーライドができること。
- **効果の可視化**: Before/After の容量・削減率が表示され、重要写真は並置で劣化を確認してから書き出せること。
- **プライバシー**: 画像がサーバーに送信されないことがファーストビューで明示されること。
- **非機能の生命線**: 20〜30枚（〜25MB目安）の一括処理でもブラウザが固まらず（Web Worker 逐次・進捗表示）、WCAG AA・レスポンシブ・キーボード操作を満たすこと。
- **言語不問**: 日英切替なしでも、アイコン＋数値主導で言語に依存せず操作できること（U-05）。
