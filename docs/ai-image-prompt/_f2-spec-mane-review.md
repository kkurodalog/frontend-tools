## 5. Mado の★要確認（⑤tex_skin と ⑦person_pores の役割分担）への判断・提言

**spec の現案（⑦に寄せる＋⑤は人物以外汎用＋両オン de-dup）を推奨。⑤への一本化はしない。**

理由:

- **被写体分岐の設計思想と整合**: §2-3 で「人物のリアリティ＝⑦不完全さに専用項目が集まる（毛穴・目の反射・非対称・年齢）」と決めている。person_pores（毛穴）はこの人物リアリティ群の一員＝**⑦に居るのが文脈的に自然**。利用者も「人物の自然さを足す」操作を⑦で一括して行える（UXの一貫性）。
- **⑤の役割が明確になる**: ⑤質感のコアは film grain＝**物・空間の素材感が主役**（texture §6-C「人物は素で可・物空間が主役」）。⑤に人物肌を残すと「⑤=物の質感」という主役定義がぼやける。**⑤tex_skin は person 既定オフのまま薄く残す**（弱いモデル向けの控えめ投入＝§3-A⑤注記どおり）か、思い切って⑤tex_skin を廃止して⑦person_pores に一本化してもよい。
- **私の最終推奨**: **⑤tex_skin は残置せず⑦person_pores に一本化**するのが最もシンプル（de-dup ロジック自体が不要になり §4-3 の実装が軽くなる・利用者の「同じことを2か所で聞かれる」混乱も消える）。ただし「弱いモデル向けに肌質感を控えめデフォルト投入したい」要件（texture §6-C）を厳密に残すなら spec 現案（両持ち＋de-dup）でもよい。**シンプルさ優先なら一本化／検証で確定した控えめ投入要件を厳密に残すなら現案**。どちらでも実装可能だが、**辞書設計を始める前に1つに決めたい**。

→ これは黒田さん確定事項（§9 のとおり）。**🟡 F3着手前に「一本化 or 現案維持」を一言確定**すれば、Cody は辞書の置き場所を迷わず実装できる。

## 6. マネタイズ整合（§2-9 広告フリー・リード本命と矛盾しないか）

**矛盾なし。** spec はツールUIに広告・寄付・課金導線を一切置かず（広告フリー）、共通枠 partial の CTA（制作者カード等）でリード動線を担う設計（§6 冒頭）。

- 過剰約束禁止コピー（§0-2／§7-3＝「近づける・組み立てる」で統一・「必ず/絶対/実写になる」禁止）は、**スキル証明＝リード価値**と本質整合。誇大ジェネレーターと一線を画すことが差別化＝リード本命の趣旨そのもの（overview §5）。
- ネガティブ別欄・参考AI注記・賞味期限管理（1箇所集約）も、誠実な事実ベースUI＝ブランド整合を崩していない。
- 軽微提言（overview評価 §4で出した動線）: 本ツール用 frontend-note 記事（記事⇄ツール相互送客）は 03_seo（Sara）の射程＝spec のスコープ外で正しく触れていない（混入なし＝良い）。
