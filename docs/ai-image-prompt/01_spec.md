## 0. 設計思想（このツールが約束すること／しないこと）

### 0-1. コア定義（変更不可・PLAN.md §2-10）

| 項目 | 確定 |
|---|---|
| 方式 | **決定論ビルダー単体**（同じ入力→同じ出力／API接続なし・LLM対話なし・乱数なし） |
| 動作 | ブラウザ完結・無料・キー不要・**入力を送信しない**（受け渡し型＝markloom型） |
| 出力 | **プロンプト文（日本語版／英語版）＋ネガティブプロンプト**の3系統。生成はしない・AIに繋がない |
| モデル | **モデル非依存の自然文＋カンマ区切り**が本体。固有記法（MJ `--ar`/`--no`・SD `(word:1.3)`）は本体に混ぜず注記で逃がす（§8） |
| スコープ | **完成形**（ミニマルにしない）＝9カテゴリ網羅。ただし二層構造（コア5問＋深掘り段階投入＝§2-2） |

### 0-2. コピー方針（過剰約束の禁止・★最重要・ブランド整合）

- **NG（断定・誇張）**: 「AI臭さを必ず消す」「これで実写になる」「絶対に自然」
- **OK（事実ベース）**: 「**自然な写真に近づけるプロンプトを組み立てます**」「AI臭さを抑える要素を織り込みます」
- 理由: モデルは入れ替わり・効きは被写体やモデル品質に依存する（実機検証で確認＝§3-1/§3-2）。断定すると公開後に裏切る。**ツールの仕事は『プロンプトを組み立てるところまで』**で言い切る。
- この方針は UI 文言・見出し・参考AI注記すべてに適用（§7-3）。

### 0-3. 実機検証で確定した「効く型」（spec の根拠・最優先反映）

| 確定 | 出典 | spec反映先 |
|---|---|---|
| **光**: 必須コア＝**光の方向＋落ちる影**（L1相当）。加える＝**柔らかい影／拡散光**（硬軟）。時間帯/天候は**任意・既定オフ**（単独は逆効果＝硬軟とセット時のみ） | lighting §6-C | §3-A（カテゴリ④） |
| **質感**: 必須コア＝**自然なノイズ／film grain**（物・素材で効く・T2相当）。素材ディテール／レンズ不完全さは**任意上乗せ・既定控えめ**。**⑤は物・素材向けに専念**（人物の肌質感は⑤に持たず⑦に一本化＝後述） | texture §6-C | §3-A（カテゴリ⑤） |
| **被写体タイプで効きが真逆**: 人物＝光主役・質感素で可／物・空間＝質感主役・光見えにくい | 両検証 §6-C | §2-3 分岐の根拠 |
| **モデル非依存ゆえ控えめにデフォルト投入**（高品質モデルで過剰にならず・弱いモデルでも効く） | texture §6-C | §3 既定値の置き方 |

## 1. データモデル全体像（決定論の核・Cody が最初に読む節）

### 1-1. 状態（answer state）→ 出力の純粋関数

ツールの本体は **「回答 state → プロンプト3文字列を導出する純粋関数」**。乱数・API・時刻・LLM を一切使わない。同じ state なら必ず同じ3文字列が出る（決定論）。

```
answerState（ユーザーの回答の集合）
   │  buildPrompt(answerState, lang)   ← 純粋関数（副作用なし）
   ├─► promptJa   : 日本語プロンプト文（カンマ／読点区切り・語順固定）
   ├─► promptEn   : 英語プロンプト文（カンマ区切り・語順固定）
   └─► negative   : ネガティブプロンプト（英語・重み記法なし・別欄）
```

- `answerState` は「カテゴリID → 選択された optionKey（複数可カテゴリは optionKey[]）」のマップ＋自由入力欄の文字列。
- ライブプレビュー＝state が変わるたび `buildPrompt` を呼び直して再描画（§6）。

### 1-2. プリセット辞書の対構造（Mane §1 唯一の実装注意＝最優先のデータ設計）

各 optionKey は **日本語文／英語文を必ず対で持つ**。ここがズレると「日本語版と英語版で言っている内容が違う」事故になる（Mane指摘）。辞書の最小スキーマ:

```ts
// 概念スキーマ（実装言語は問わない。構造を守ること）
type PresetOption = {
  key: string;          // 一意キー（answerState が参照）
  labelJa: string;      // UI表示ラベル（日本語）
  fragmentJa: string;   // 日本語プロンプトに連結する語句
  fragmentEn: string;   // 英語プロンプトに連結する語句（labelJa と対）
  subject?: SubjectType[]; // この選択肢が出る被写体タイプ（分岐・§2-3）。未指定=全タイプ共通
  group: CategoryId;    // 所属カテゴリ（語順制御に使う・§4）
  default?: boolean;    // 既定選択か（控えめにデフォルト投入＝§0-3）
};
type SubjectType = "person" | "object" | "scene"; // 人物 / 物 / 空間（風景含む）
```

- **辞書の種は `_research-knowledge-base.md` の日英語彙表＋本書 §3 のプリセット表**をそのまま使う（机上の語彙が一次ソース／光・質感は実機確定値で上書き済み）。
- `fragmentJa`／`fragmentEn` は **語句断片**（文ではなく連結部品）。`buildPrompt` がカテゴリ順に並べてカンマ／読点でつなぐ（§4）。
- ネガティブは別辞書（§3-A カテゴリ⑧／§5）。英語のみ・重み記法なし。

## 2. 質問フロー（9カテゴリ＋二層構造＋被写体分岐）

### 2-1. 9カテゴリ一覧（overview §4-1 の完成形を確定）

| ID | カテゴリ | 何を聞くか | 層（§2-2） | 被写体分岐（§2-3） |
|---|---|---|---|---|
| ① | **画像タイプ** | 写真／イラスト／3D／抽象＋Web用途（ヒーロー/背景/アイキャッチ/人物/商品/装飾） | **コア** | なし（全タイプ共通の入口） |
| ② | **被写体・主題** | 人物／物／空間（風景）＋具体物の自由入力。**ここで被写体タイプが決まり以降を分岐** | **コア** | **ここで分岐軸を決定** |
| ③ | **雰囲気・トーン** | 与えたい印象（落ち着いた/親しみ/高級感 等）・色味（彩度控えめ等） | **コア** | 共通（語彙は同じ） |
| ④ | **光（ライティング）** | 光の方向／影の柔らかさ（柔/硬）／【任意】時間帯・天候 | **コア** | **人物で厚く効く**（既定オン）／物・空間は弱め（既定はコアのみ） |
| ⑤ | **質感・素材感** | 自然なノイズ/film grain／【任意】素材ディテール・レンズ不完全さ（**物・素材向けに専念＝肌質感は持たない**） | **深掘り** | **物・空間で厚く効く**（既定オン）／人物は任意（film grainのみ） |
| ⑥ | **構図・カメラ** | 被写体位置（左/右/中央）・三分割・余白／【任意】アングル・焦点距離・被写界深度 | **コア（位置のみ）** | 共通（center bias 対策は全タイプ） |
| ⑦ | **不完全さ／リアリティ** | candid（不意の瞬間）・生活感／人物時のみ＝毛穴・目の反射・非対称・年齢 | **深掘り** | **人物で専用項目が増える**（§3-A 参照） |
| ⑧ | **避けたい要素（ネガティブ）** | 入れたくない見た目（CG臭・指破綻・透かし 等）。別欄生成 | **深掘り** | 人物選択時に解剖系を厚く（§5） |
| ⑨ | **アスペクト比** | 16:9／1:1／縦長 等（Web用途デフォルト） | **深掘り** | なし（用途デフォルト） |

### 2-2. 二層構造（Mane 条件②＝工数の天井・降格ライン）

完成形9カテゴリを**コア5問（v1必達）＋深掘り4段（段階投入）**に割る。コア5問だけでプロンプトが成立し、深掘りは「詳しく詰める」任意ステップ。

| 層 | カテゴリ | 役割 | v1での扱い |
|---|---|---|---|
| **コア5問**（必達） | ①画像タイプ・②被写体・③雰囲気・④光・⑥構図（被写体位置まで） | これだけで「自然に近づくプロンプト」が完成する最小セット | **v1必達**。ここが通れば公開可 |
| **深掘り4段**（段階投入） | ⑤質感・⑦不完全さ・⑧ネガティブ詳細・⑨アスペクト比 | 効きを上乗せ・精度を上げる任意ステップ | 知識ベースの仕上がりに応じて段階投入。工数オーバー時は簡素化可 |

- **実装順序（Mane §3-3 提言）**: コア5問のフロー＋プレビュー＋3系統コピー機構を先に通す（辞書がプレースホルダでも UI が動く構造）。深掘り段は後から辞書を埋めて有効化。
- UIでは「**ここまでで完成（コア）／さらに詰める（深掘り）**」の区切りを明示（§6-1）。深掘りに進まなくても完成プロンプトは出る。

### 2-3. 被写体タイプ分岐（実機検証の最重要設計）

**②被写体で `person / object / scene` のどれかが確定**し、以降のカテゴリの**既定・出す選択肢・厚みが変わる**。これは「人物＝光主役／物・空間＝質感主役」という実機検証の対称性（lighting/texture §6-C）を質問フローに落とすもの。

| カテゴリ | person（人物） | object（物） | scene（空間・風景） |
|---|---|---|---|
| ④光 | **厚く効く＝既定オン**（方向＋影＋硬軟） | 弱め（方向＋影のコアのみ・硬軟は任意） | 弱め（同左） |
| ⑤質感 | **控えめ／任意**（film grainのみ・肌質感は⑤に持たず⑦で扱う） | **厚く効く＝既定オン**（film grain＋素材ディテール） | **厚く効く＝既定オン**（film grain＋素材感） |
| ⑦不完全さ | **専用項目あり**（毛穴・目の反射・非対称・candid表情・年齢） | candid/生活感のみ | 生活感・環境の文脈（散らかり等） |
| ⑧ネガティブ | **解剖系を厚く**（指破綻・bad anatomy・dead eyes） | 物体破綻寄り（軽め） | 背景連続性・文字崩壊寄り |

> 実装: 分岐は `PresetOption.subject` で制御（§1-2）。`buildPrompt` と UI は `answerState[②]` の subject 値で「出す選択肢・既定 ON/OFF」を切り替える。**決定論は崩れない**（subject も answerState の一部＝同じ入力なら同じ分岐）。

## 3. カテゴリ別プリセット表（辞書の種＝Cody が辞書化する元データ）

> 各カテゴリの **選択肢・日英断片・既定値・被写体分岐・v1本数の天井** を確定する。
> 光④・質感⑤は**実機検証 §6-C の確定値**を反映（机上知識ベースより優先）。他は知識ベースを基に机上で設計。
> **v1本数の天井**＝各カテゴリで v1 に持つ選択肢の上限（Mane 条件②＝縦の深さの青天井化を防ぐ）。盛りすぎず保守可能な範囲。

| `light_right` | 右から | 右側からの光、左側にできる影 | light coming from the right, shadows on the left | |
| `light_back` | 逆光 | 被写体の背後からの光（逆光） | backlight from behind the subject | |
| `light_top` | 上から | 上方からの自然光 | natural light from above | |

|---|---|---|---|

> **分岐**: `light_soft` は **person で既定オン**・object/scene では任意（§2-3）。`time_*` は全タイプで既定オフ。

#### ⑤ 質感・素材感（深掘り／実機確定＝texture §6-C 反映・v1天井=コア2＋素材2＝物・素材向けに専念）

> **確定型**: 必須コア＝**自然なノイズ／film grain**（物・素材で S9 過剰クリーンを消す）。素材ディテール／レンズ不完全さは**任意上乗せ・既定控えめ**（T2で足りる）。**⑤は物・素材向けの質感に専念し、人物の肌質感は持たない**（人物の肌は⑦`person_pores`に一本化＝高品質モデルは素で十分・盛りすぎ注意のため⑦側で任意扱い）。

## 4. プロンプト構築ロジック（決定論＝同じ入力→同じ出力）

### 4-1. 語順（固定・カテゴリ順に連結）

`buildPrompt` は **下記の固定順序**で各カテゴリの選択断片を並べる。これにより同じ state なら必ず同じ語順になる（決定論）。順序は実機検証キットの固定ベース文（lighting/texture §1）の並びに準拠＝「写真宣言→被写体→構図→カメラ→光→質感→雰囲気→不完全さ」。

```
1. ①画像タイプ（写真宣言）        例: 写真、実写 / RAW photo, photorealistic
2. ②被写体（自由入力含む）        例: 30代の男性…/ a man in his 30s…
3. ⑥構図（位置→余白→焦点→絞り）  例: 三分割、左1/3、35mm、浅い被写界深度
4. ④光（方向→硬軟→時間帯）        例: 左からの光、柔らかい影、ゴールデンアワー
5. ⑤質感（grain→素材→レンズ）      例: フィルムの粒状感、素材ディテール
6. ③雰囲気・トーン（印象→色味）    例: 落ち着いた雰囲気、彩度控えめ
7. ⑦不完全さ（candid→生活感→人物）例: スナップ風、毛穴、目の反射
```

- **⑧ネガティブは本文に入れない**（別欄＝§5）。**⑨アスペクト比も本文に入れない**（注記＝§8）。
- 各断片は **カンマ（英語）／読点（日本語）** で連結。末尾の写真メディア語（`shot on a 35mm lens` 等）はカメラ群にまとめる。

### 4-2. 連結規則

- 区切り: 英語＝`, `（カンマ＋半角スペース）／日本語＝`、`（読点）。
- 文の開始＝画像タイプ（写真宣言）を必ず先頭に。被写体の自由入力はその直後。
- 空のカテゴリ（未選択）は**スキップ**（区切り文字を二重に出さない）。
- 自由入力欄の文字列は**トリムして差し込むのみ**（加工しない＝ユーザー入力をそのまま尊重・決定論）。

### 4-3. ライブプレビューの挙動（overview §4-2）

- state が変わるたび `buildPrompt(state,'ja')`・`buildPrompt(state,'en')`・`buildNegative(state)` を再計算しプレビュー再描画。
- **答えるほど文章が育つ**（コア5問時点で完成文・深掘りで語が増える）。途中でも常に「今の完成形」が見える。
- 純粋関数なので副作用なし・再描画コスト小（Mane §1 実現性確認済み）。

### 4-4. 決定論の保証（自己検証ポイント）

- 乱数・`Date.now()`・API・LLM を `buildPrompt` 系で**一切使わない**。
- 同じ `answerState`（被写体タイプ・全選択・自由入力文字列が同一）→ 同じ3文字列が**必ず**返る。
- これがツールの信頼性の核（再現性＝コピーして使い回せる）。

## 5. ネガティブプロンプト生成（別欄・英語・重み記法なし＝★3）

- `buildNegative(state)` は §3-A ⑧の `negativeEn` を**カンマ区切りで連結**（重み記法 `:1.3` を付けない＝★3 確定）。
- 被写体が `person` のとき `neg_skin`・`neg_face` を既定/任意で追加（§2-3）。写真タイプのとき `neg_photo` を既定追加。
- 出力は**独立した欄**に表示し、**独立コピー**ボタンを持つ（§6-3）。
- **注記（必須・ネガティブ欄直下）**: 「ネガティブプロンプトは**対応するAIでのみ有効**です（記法を持たないAIもあります）。Midjourney は `--no`、Stable Diffusion 系はネガティブ欄に貼ってください。」← overview ★E／知識ベース §7 を文言化。

## 6. UI 設計（共通枠＝design-concept.md の世界観に乗せる）

> ヘッダー/フッター/パンくず/制作者カード/CTA は共通枠（partial）が担う。本書はツール本体エリア（`--tool-panel-*` の器）の中身を扱う。第1弾 `image-compress` と同じく差分トークン（`--tool-panel-*`/`--field-*`）を使う。

### 6-1. 画面フロー（ステップ式・コア→深掘り）

1. **コアステップ（①〜④・⑥位置）**: 1画面1〜数問のステップ式（前へ/次へ）。②被写体を選んだ時点で以降が被写体分岐（§2-3）。右側（またはプレビュー領域）に**生成プロンプトがライブ表示**（答えるほど育つ）。
2. **「ここまでで完成」区切り**: コア5問完了で**完成プロンプトが揃う**。「これで使えます（コピー）／さらに詳しく詰める」の分岐を明示。深掘りに進まなくてもコピー可能。
3. **深掘りステップ（⑤⑦⑧⑨）**: 「詳しく詰める」を選んだ時のみ表示。質感・不完全さ・ネガティブ詳細・アスペクト比を任意で追加。各深掘り項目に**既定値＋注記**（§3-A の UI注記）。
4. **受け渡し（完成エリア）**: §6-2/§6-3。

### 6-2. プレビュー領域（ライブ）

- **日本語版／英語版を切替表示**（トグル1つ・出力プロンプト言語のみ＝overview §4-3）。**初期値＝英語**（画像AIは英語に強い）。日本語版も即切替で見られる。
- **★C 表示密度（overview ★C）の確定**: **「英語を主表示＋トグルで日本語」**を採用（常時併記は出力欄が縦に伸びすぎるため）。トグルは目立つ位置に置き、日本語版も1クリックで全文見える。
- プレビューはステップ進行中も常時表示（育つ様子が見える）。

### 6-3. 受け渡しUI（完成プロンプトの提示・独立コピー）

- 完成プロンプトを提示し、**3系統を独立してコピー**:
  1. 日本語版プロンプト（コピー）
  2. 英語版プロンプト（コピー）
  3. ネガティブプロンプト（コピー・§5 注記付き）
- コピー成功フィードバック（第1弾の実装済み資産を流用＝Mane §1）。
- **参考AI注記（完成プロンプト直下・控えめなノート・確定文言）**:
  > 「このプロンプトは特定のAI専用ではありません。画像生成に強いAIの例（**2026年時点**）: Flux 2 / Imagen 4 / GPT Image 2 など。モデルは入れ替わるため、お使いのAIに合わせて調整してください。」
  - **年付き・例示・断定しない**書き方を固定（overview §4-2／§0-2 過剰約束禁止）。
- **★D 賞味期限管理**: 参考AI注記のモデル例と「2026年時点」は**1箇所の定数／1ファイル**に集約し、更新を1点修正で済む構造にする（Mane ★D／overview ★D）。「いつ誰が見直すか」は公開後 todo 化。

### 6-4. ★2 機種名/フィルム銘柄＝任意トグル（必須コアに入れない）

- 機種名（`shot on Canon EOS R5` 等）・フィルム銘柄（`Kodak Portra 400` 等）は**深掘りの最下層に任意トグル**で置く（既定オフ）。
- 知識ベース §3-A/§3-C で「効果あり」とされるが**固有性が高くモデル非依存原則とぶつかる**（★2）。よって**必須コアには入れず・上乗せ扱い**。
- トグル注記: 「カメラ機種名・フィルム名を加えると光学的な質感が乗ることがありますが、AIによって効きが変わります（任意）。」

### 6-5. 言語不問UIへの配慮（第1弾 §7 と整合）

- UI ラベルは日本語ベースでよい（i18n 全面採用は v1 でしない＝overview §4-3）。
- ただし各カテゴリにアイコンを添え、選択肢が言語に頼らず識別できるようにする（第1弾の言語不問UI思想を踏襲・必須ではないが推奨）。

## 7. コピー・文言方針（対外UI・過剰約束の禁止）

### 7-1. 見出し・説明文（事実ベース）

- ツール説明（例）: 「**Web制作で使う画像を、自然な写真に近づけるプロンプトを組み立てます。**」（§0-2）
- カテゴリ見出しは平易に（「光の方向」「背景をぼかす」等＝専門用語を避け具体で）。第1弾 §1-2 と同じ粒度。

### 7-2. 注記の標準セット（必ず出す）

| 場所 | 文言の趣旨 |
|---|---|
| 時間帯/天候欄（④） | 単独だと演出過剰・硬軟とセット推奨（既定オフ）＝§3-A ④ |
| 人物の肌質感欄（⑦`person_pores`） | 高品質AIは素で十分・気になる時だけ＝§3-A ⑦ |
| ネガティブ欄（⑧） | 対応AIのみ有効・MJ/SDの貼り方＝§5 |
| アスペクト比（⑨） | お使いのAIの記法で値を指定＝§8 |
| 完成プロンプト直下 | 参考AI注記（年付き・非断定）＝§6-3 |

### 7-3. 禁止表現（§0-2 の徹底）

- 「必ず」「絶対」「AI臭さを消す（断定）」「実写になる」を**使わない**。
- 「近づける」「抑える」「織り込む」「組み立てる」で統一。

## 8. モデル固有記法の扱い（知識ベース §7・モデル非依存原則）

- **本体プロンプト＝モデル非依存の自然文＋カンマ区切り**（§4）。MJ `--ar`/`--no`・SD `(word:1.3)` は**本文に入れない**。
- **アスペクト比（⑨）・ネガティブ（⑧）は「お使いのAIの記法で」と注記で逃がす**:
  - Midjourney: 比率は `--ar 16:9`、ネガティブは `--no ...`
  - Flux: 比率は width/height（実数）、MJ記法は付けない（渡すと劣化）。本文の自然文がそのまま向く
  - Imagen / GPT Image: 比率・ネガティブは UI/パラメータ側。本文は自然文でOK
- この注記は §6-3／§5／§3-A ⑨ に分散配置（値だけ出す＝記法はユーザー側）。

## 9. ★要確認・実機調整事項

- **確定（黒田さん＋Mane）**: 人物の肌質感は **⑦`person_pores` に一本化**（⑤質感側には肌質感を持たせない）。二重定義を排し、実装の de-dup を不要にしてシンプル化する。⑤は物・素材向けの質感（grain／素材ディテール／レンズ不完全さ）に専念（§3-A ⑤・⑦）。← 旧 ★要確認（⑤/⑦の役割分担）はこの決定でクローズ。
- **★要確認（公開前・任意検証）**: イラスト/3D 選択時のAI臭消し語彙は知識ベースで薄い（§3-A ①／知識ベース §8-2）。**v1は写真主軸**で出し、イラスト系は film grain 等の写真前提語を出さない設計にした。イラスト専用プリセットが要るかは公開後の反応で判断（v1は持たない）。
- **★要確認（確度の注記・両検証 §6-C）**: 光・質感の確定型は**単一モデル（Imagen 4 / Google Flow）・少数枚・主観評価**＝確度は中程度。複数モデルでの再確認は今後の任意課題（spec はこの確度前提で書いている）。
- **実機調整（F3後）**: 各カテゴリのプリセット文面が「実際に効く」かは、F3で UI が動いた後に黒田さんが実生成で当て直す余地を残す（第1弾の机上→実機書き換え教訓／Mane §3）。辞書は差し替え可能な構造（§1-2）にしてあるため、文面調整は辞書の値変更で済む。
