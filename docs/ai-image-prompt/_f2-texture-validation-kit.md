# _f2-texture-validation-kit — 「質感」カテゴリ 実機検証キット

> F2 spec前段の縦スライス検証フェーズ第2弾（担当: Mado / 制作部）。光カテゴリ検証完了（`_f2-lighting-validation-kit.md` §6-C 確定）を受けた横展開。
> **目的**: 知識ベース（`_research-knowledge-base.md` §3-C 質感・フィルム/粒状感）はすべてWeb記事ベース＝実機未検証。机上では「film grain を足せばデジタル臭が消える」が**最も多くのソースで一致した手法**だが、光検証で「机上推奨（時間帯/天候単独）が実機では逆効果」だった前例がある。よって**「質感」1カテゴリだけを縦スライスで実機検証**し、"効く型" を確定してから残りカテゴリへ横展開する。
> **検証の狙い**: AI臭 **S1「プラスチック肌/のっぺり質感」**・**S9「過剰なツルツル/デジタル臭」**（知識ベース §2）を、質感の指定（**フィルム粒状感→ノイズ/アナログの揺らぎ→素材ディテール/レンズの不完全さ**の段階／§3-C）で消せるかを実機で当てる。
> **このキットの正体**: 黒田さんが画像生成AIに貼って比較する「宿題セット」。T0〜T3を出して並べ、§4の判定軸で「どの段階が効くか」を判定→§6の結果テーブルに書き戻す。それが埋まれば質感プリセットが確定し、同じ型を構図・人物へ横展開できる。

---

## 0. 検証設計の前提（変更不可・知識ベース／光キットと整合）

- **受け渡し型**: ツールはプロンプト生成まで／生成はユーザー。よって本キットも「コピペして各自のAIで出す」形。
- **モデル非依存の自然文＋カンマ区切り語彙が本体**。Midjourney `--ar`/`--no`・SD重み記法 `(word:1.3)` は**混ぜない**（知識ベース §7）。
- **★2 機種名/フィルム銘柄（Kodak Portra 等）＝任意トグル**。本検証の固定ベースにも段階バリエーションにも**入れない**（質感の核＝粒状感/ノイズ/素材感だけを切り分けるため。フィルム銘柄は別の任意上乗せ検証で扱う）。
- **★3 ネガティブ重み記法 `:1.3`＝汎用版は重み無し**。本キットのベースにもネガティブ自体を入れない（質感の効果だけを切り分けるため。`plastic skin` 等のネガティブ検証は段8の別カテゴリで実施）。
- **光は確定型で固定（変数にしない）**: 光検証で確定した「**方向＋落ちる影＋柔らかい影/拡散光**」（光キット §6-C の必須コア＋硬軟）を**固定ベースに含める**。時間帯/天候（ゴールデンアワー等）は単独だと逆効果だったため**入れない**。自然な光の上で質感の効きを見る。
- **縦スライスの鉄則**: 被写体・構図・光（確定型）・カメラ基本は**全段階で完全固定**。変えるのは**質感の指定だけ**。これが崩れると「質感が効いたのか他が効いたのか」が切り分けられない。

## 1. 固定ベースプロンプト（質感以外＝全段階共通の土台）

各検証題で、T0〜T3を通して**この土台は一字一句変えない**。質感の語だけを段階ごとに追記する。
土台に含めるのは「写真宣言＋具体被写体＋構図＋**光の確定型**＋カメラ基本（焦点距離・被写界深度）」のみ。**質感に関する語（film grain・粒状感・ノイズ・素材ディテール・レンズの不完全さ等）は土台に一切入れない**（T0を純粋な対照群＝S1/S9が出やすいAI任せ状態にするため）。

| 検証題        | 被写体（光キットから流用＝Web制作で使う実写）                                | Web用途の想定                               | 質感で見たい核                                |
| ------------- | ---------------------------------------------------------------------------- | ------------------------------------------- | --------------------------------------------- |
| **題A** | 人物ヒーロー系（30代の男性フリーランサーがノートPCの前で自然な表情）         | サービスサイト/LPのヒーロー・人物セクション | **S1 プラスチック肌**（肌の質感）       |
| **題B** | 室内・物・空間系（木製デスクの上のコーヒーカップと観葉植物のある作業デスク） | 制作実績/ブログのアイキャッチ・背景・装飾   | **S9 過剰クリーン**（物の素材グレイン） |

> 注: 題Aの「30代男性」「自然な表情」は被写体記述として固定に含む。**人物リアリティ語（毛穴/非対称/目の反射等）は土台にも段階にも入れない**——それは段2/7の人物カテゴリ検証で扱う。本キットで段階追加する「肌の質感」は**汎用の質感（粒状感→ノイズ→素材ディテール）の文脈で肌に触れる範囲**に留め、人物専用のリアリティ語とは切り分ける。
> 注: 光の語は土台に固定で入る（**方向＋落ちる影＋柔らかい影/拡散光**）。これは光検証で確定済みの「効く型」なので、質感検証では変数化しない。

## 4. 判定軸（黒田さんが各段階を「効いた/効かない」で見る観点）

各段階の出力画像を、以下のチェックリストで見る。**目的はAI臭 S1「プラスチック肌/のっぺり」・S9「過剰クリーン/デジタル臭」が消えたかを具体的に見ること**。各項目を「はい/いいえ/どちらとも」で判定。

| #  | チェック項目（S1/S9が消えたか）                            | 見るポイント                                                                                                  |
| -- | ---------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| C1 | **肌/素材に微細な質感があるか（S1対策）**            | 題A=肌に毛穴/微細な凹凸が見えるか・テカテカのエアブラシ肌でないか。題B=木目/陶器/葉に表面のディテールがあるか |
| C2 | **ツルツルすぎないか（S1対策）**                     | 表面が異様に滑らかでのっぺりしていないか。プラスチック/ワックスのような人工的な質感でないか                   |
| C3 | **粒状感が自然か（S9対策）**                         | フィルムの粒状感が画面全体に自然に乗っているか。粒が無く過剰にクリーンでないか                                |
| C4 | **過剰なノイズで汚くなっていないか（やりすぎ検出）** | 粒状感/ノイズが強すぎてザラザラ・汚い・低画質に見えていないか（T2/T3で特に注意）                              |
| C5 | **"いかにもAI"なクリーンさが減ったか（S9総合）**     | 一目で「AI生成だな」と分かる無菌的なツルッとしたデジタル臭が、段階を上げて減ったと感じるか                    |
| C6 | **質感以外が崩れていないか（縦スライス確認）**       | 質感語を足したことで構図/光/被写体が変わっていないか（変わっていたら比較が成立しない＝シードや表現を見直す）  |

### 段階間の比較メモ（自由記述・どの段階が最も自然だったか）

| 検証題          | T0（なし）                                                                                                                                                                | T1（粒状感） | T2（＋ノイズ/揺らぎ） | T3（＋素材/レンズ・フル） |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------ | --------------------- | ------------------------- |
| 題A（人物・肌） | 全体的に品質良い。T0~T3で違いは見受けられない。                                                                                                                           | 同左         | 同左                  | 同左                      |
| 題B（物・素材） | 全体的に品質良い。それを前提とするが、題BについてはT2,T3の画像が最も良い。T0,T1はややツルッとしたテクスチャをしている。（それでも非常に品質は良いので微妙な差でしかない） | 同左         | 同左                  | 同左                      |

> 記入例: 各セルに「S1/S9残る／少し改善／自然／やりすぎで汚い」等の所感や、C1〜C6で気づいた点を短くメモ。

## 5. 実施手順（黒田さん向け・Google Flow / Imagen 4）

1. **Google Flow（Imagen 4）で実施**（光検証と同じ環境＝モデル差の影響を排除。無料・フォトリアル）。
2. **題Aから開始**。T0→T1→T2→T3 の順で生成する。
3. **シード固定UIが弱い前提**: Google Flow はシード固定が弱いので、**各段階を2〜3枚ずつ出して傾向で見る**（1枚の当たり外れで判断しない）。同じ段階の2〜3枚を見て「この段階は概してこう」という傾向を掴む。
4. 出てきた各段階の代表画像を**横に並べて**、§4の判定軸（C1〜C6）で見比べる。特に**T0（質感なし）とT1〜T3の差**を見る（質感を足して S1/S9 が減ったか）。
5. **題Bでも同じことを繰り返す**（題Aは肌＝S1、題Bは物の素材＝S9 が主眼）。
6. 気づいたことを §6 の結果テーブルに書き戻す。**特に「机上推奨（film grain を足せば効く）が実機で効いたか/効かなかったか」を §6-C に正直に記録する**（光でL2が逆効果だった前例があるため）。

### モデル固有記法の注意（知識ベース §7）

- **このキットのプロンプトはモデル非依存の自然文**です。Midjourney の `--ar`/`--no`、Stable Diffusion の `(word:1.3)` のような**固有記法は含めていません**。そのまま貼ってください。
- **アスペクト比・ネガティブ**は、Google Flow の UI 側で指定してください（プロンプト文には書いていません）。比率は全段階で揃えること。
- **フィルム銘柄（Kodak Portra 等）は今回は入れていません**（質感の核＝粒状感/ノイズ/素材感だけを切り分けるため）。効きを見たくなったら別途、確定した質感型の上に任意で足して試す。

### 6-C. spec への申し送り（黒田さん→Mado / 2026-06-26 黒田さん検証＝Google Flow/Imagen 4・チーフ転記＝黒田さん合意済）

- **質感プリセットの確定型**:
  - **必須コア＝自然なノイズ／film grain**（T2相当）。**物・素材**で S9（過剰クリーン/デジタル臭）を消す方向に効く。
  - **加える＝素材ディテール／レンズの不完全さ**（T3）は**任意の上乗せ・既定は控えめ**（T2で足りるため）。
  - **避ける/任意オフ＝人物への厚い質感**。Imagen 4 のような高品質モデルでは人物の肌は素で十分＝質感を盛っても差が出ない／盛りすぎ注意。
- **質問フローで聞くべき粒度**:
  - **被写体タイプで分岐**させる（最重要）。
  - 「**質感（ノイズ/film grain）**」は**物・空間で聞く＝採用**（コア）。
  - 「**素材ディテール**」を物で任意に聞く。
  - 「**人物の肌質感**」は**控えめ/任意**（高品質モデルでは不要・低品質モデル向けに弱くデフォルト投入）。
- **★机上の推奨が実機で効いたか/効かなかったか**:
  - 知識ベース §3-C「film grain / natural noise でデジタル臭を消す」（最多ソース一致の手法）→ **物・素材では効いた（T2以降）／人物（高品質モデル）では差が見えなかった**。＝**被写体とモデル品質に依存**（逆効果ではない）。
  - 光検証ほど明確な"机上と実機の食い違い"はなかったが、**「効くが見えにくい領域（人物×高品質モデル）」がある**ことが判明。モデル非依存ツールとして**弱いモデルでも効くよう控えめにデフォルト投入**する設計に倒す。
- **被写体差**: 大きい。**人物＝"光"が主役・質感は素で可／物・空間＝"質感（ノイズ・素材感）"が主役・光は見えにくい**（光検証と真逆の対称性）。spec の質問フローを被写体タイプで分岐させる強い根拠。
- **横展開の指示**: **これ以上の縦スライス実機検証は行わず、残りカテゴリ（画像タイプ・被写体・雰囲気・構図・不完全さ・ネガティブ・アスペクト比）は知識ベースを基に Mado が机上で spec 化する**（黒田さん決定 2026-06-26＝光・質感で効く型と検証手法が掴めたため）。必要箇所だけ公開前に追加検証。
- **確度の注記**: 単一モデル（Imagen 4 / Google Flow）・少数枚・主観評価のため確度は中程度。複数モデルでの再確認は今後の任意課題。
